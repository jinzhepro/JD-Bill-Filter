    end

    Logic --> Data
    
    Pages --> SettlementCtx
    Pages --> InvoiceCtx
    SettlementCtx --> Processor

**架构说明**：
- **UI Layer**：只负责展示和交互，不包含业务逻辑
- **State Management**：管理应用状态，统一数据流（单向数据流）
- **Business Logic**：处理核心业务算法，独立可测试
- **Data Layer**：数据持久化，Edge Runtime 运行

## 图表二：结算单处理流程图

```mermaid
flowchart TD
    Start([原始结算单数据]) --> Validate{数据验证}
    
    Validate -->|失败| Error1[抛出错误:<br/>缺少必需列]
    Validate -->|成功| Filter1[过滤货款记录<br/>feeCode === 30]
    
    Filter1 --> CollectFee[收集服务费数据]
    CollectFee --> DirectFee[直营服务费<br/>按 SKU 分组]
    CollectFee --> TransFee[交易服务费<br/>按 SKU 分组]
    
    DirectFee --> MergePayment[合并货款记录]
    TransFee --> MergePayment
    
    MergePayment --> GroupSKU[相同 SKU 分组]
    GroupSKU --> AccumulateAmount[累加金额<br/>使用 Decimal.js]
    GroupSKU --> AccumulateQty[累加数量]
    
    AccumulateAmount --> CalcCompensation[计算售后赔付费]
    AccumulateQty --> CalcCompensation
    
    CalcCompensation --> TotalComp[赔付费总额]
    TotalComp --> CalcRatio[计算货款占比]
    CalcRatio --> AllocateComp[按比例分摊赔付费]
    
    AllocateComp --> FinalCalc[最终计算]
    FinalCalc --> PaymentAmount[货款 = 应结金额 - 分摊赔付费]
    FinalCalc --> Revenue[收入 = 货款 + 直营服务费]
    
    PaymentAmount --> Export[导出 Excel]
    Revenue --> Export
    
    Export --> End([处理完成])
    
    Error1 --> End
    
    style Start fill:#e1f5ff
    style End fill:#e1ffe1
    style Error1 fill:#ffe1e1
    style AccumulateAmount fill:#fff4e1
    style FinalCalc fill:#f5e1ff
```

**流程说明**：
1. **数据验证**：检查必需列（商品编号、金额列）
2. **过滤货款**：只处理费用名称为"货款"的记录
3. **服务费收集**：单独统计直营服务费和交易服务费
4. **合并货款**：相同 SKU 的金额和数量累加
5. **赔付费分摊**：按货款比例分配到每个 SKU
6. **最终计算**：计算货款和收入
7. **导出结果**：生成 Excel 文件

## 图表四：数据流架构图

```mermaid
sequenceDiagram
    participant User as 用户
    participant UI as UI Components
    participant Context as React Context
    participant Logic as Business Logic
    participant API as Edge API Routes
    participant D1 as Cloudflare D1
    
    User->>UI: 上传文件
    UI->>Context: dispatch(SET_FILE)
    Context->>Logic: 处理文件
    Logic->>Logic: 解析 Excel/CSV
    Logic->>Logic: 执行结算算法
    Logic->>Context: 返回结果
    Context->>UI: 更新状态
    UI->>User: 展示处理结果
    
    User->>UI: 导出结果
    UI->>Logic: 导出 Excel
    Logic->>Logic: 格式化数据
    Logic->>User: 下载文件
    
    User->>UI: 保存发票历史
    UI->>API: POST /api/invoice-history
    API->>D1: INSERT 记录
    D1->>API: 返回成功
    API->>UI: 响应结果
    UI->>User: 提示保存成功
```

**数据流说明**：
- 文件处理主要在 Client Side（浏览器端）
- 数据持久化通过 Edge API Routes
- Context 作为中间层协调 UI 和 Logic

## 图表五：Edge Runtime 限制与应对

```mermaid
graph TB
    subgraph Problems["Edge Runtime 限制"]
        NoFS["❌ 不能使用 fs 模块<br/>无法访问文件系统"]
        NoNodeAPI["❌ 部分 npm 包不兼容<br/>依赖 Node.js API"]
        TimeLimit["❌ 执行时间限制<br/>不能长时间阻塞"]
    end
    
    subgraph Solutions["应对策略"]
        FormData["✅ 使用 FormData API<br/>浏览器原生 API"]
        ArrayBuffer["✅ ArrayBuffer + Buffer<br/>ExcelJS 兼容方案"]
        BatchAPI["✅ API 批量操作<br/>减少请求次数"]
        EdgeExcelJS["✅ ExcelJS Edge 配置<br/>避免 fs 依赖"]
    end
    
    NoFS --> FormData
    NoFS --> ArrayBuffer
    NoNodeAPI --> EdgeExcelJS
    TimeLimit --> BatchAPI
    
    style Problems fill:#ffe1e1
    style Solutions fill:#e1ffe1
```

**应对策略说明**：
- Edge Runtime 限制需要采用不同的技术方案
- FormData + ArrayBuffer 替代 fs 模块
- ExcelJS 通过配置兼容 Edge Runtime
- 批量 API 操作提升性能

## 使用建议

### 在掘金平台使用

掘金支持 Mermaid 图表渲染，直接复制上面的代码块即可。

### 在思否平台使用

思否也支持 Mermaid，复制代码块到 Markdown 编辑器即可。

### 在个人博客使用

如果你的博客不支持 Mermaid，可以：
1. 使用在线工具渲染（如 [Mermaid Live Editor](https://mermaid.live/)）
2. 导出为 SVG/PNG 图片
3. 或者使用上面的 ASCII 艺术图版本
